# 历史日线回填方案

## 背景

当前 `daily_quote` 表只有最近一天的行情快照（A股 5,479 条、港股 2,642 条），没有历史日线数据。缺少历史日线意味着：
- 无法计算均线（MA5/MA20/MA60）、波动率等技术指标
- 无法做历史回测
- 无法观察价格趋势和成交量变化

## 数据源评估

### 方案 A：腾讯历史 K 线接口（推荐主源）

- **接口：** `https://web.ifzq.gtimg.cn/appstock/app/fqkline/get`
- **参数：** `{code},day,{start},{end},{k},qfq`
- **速度：** ~83ms/只，直连无需代理
- **限制：** 单次最多 ~800 条（约 3.2 年日 K），不支持批量（需逐只请求）
- **复权：** 支持 qfq（前复权）
- **字段：** 日期、开盘、收盘、最高、最低、成交量、成交额
- **风险：** 已验证可用，但连续大量请求可能触发限流

### 方案 B：akshare（东方财富底座，备选）

- **接口：** `ak.stock_zh_a_hist()` / `ak.stock_hk_hist()`
- **速度：** ~370ms/只（A 股），~570ms/只（港股）
- **限制：** 底层调东方财富，已有 IP 被封记录
- **优点：** 已有现成代码（`fetch_a_hist` / `fetch_hk_hist` + transform 函数 + `_backfill_hist`）
- **风险：** 大量请求可能再次被封 IP

### 方案 C：混合策略（推荐）

| 市场 | 主源 | 备选 |
|------|------|------|
| A 股 | 腾讯 K 线 | akshare |
| 港股 | 腾讯 K 线 | akshare |
| 美股 | 待调研 | — |

**策略：** 优先用腾讯接口（更快、更稳定），失败时 fallback 到 akshare。腾讯单次最多 800 条，超过的部分需分多段请求。

## 回填范围

| 市场 | 股票数 | 回填起始日 | 预计交易日数/只 |
|------|--------|-----------|----------------|
| A 股 | ~5,400 | 2021-01-04 | ~1,200 |
| 港股 | ~2,600 | 2021-01-04 | ~1,200 |
| 美股 | 518 | 待定 | — |

**暂不考虑 2021 年之前的数据**（如需更早可以后续扩展）。

## 实现方案

### 腾讯 K 线接口字段映射

```
腾讯返回格式: ['2026-04-07', '1460.05', '1440.02', '1470.00', '1435.05', '24152.00']
                日期         开盘      收盘      最高      最低      成交量
```

注意：腾讯返回的是 **6 列**（日期、开、收、高、低、量），没有成交额和换手率。

### 分段回填策略

腾讯单次最多 ~800 条（约 3.2 年）。2020 年至今约 6 年 = ~1,500 条，需分 2 段：
- 第 1 段：2020-01-02 ~ 2023-06-30（~850 条）
- 第 2 段：2023-07-01 ~ 至今（~650 条）

### 增量更新策略

- 查询 `daily_quote` 中每只股票的最新 `trade_date`
- 只拉取最新日期之后的数据
- 如果已是最新，跳过
- 这样后续每天只需拉增量，不需要全量重跑

### 代码实现

1. **新增腾讯 K 线 fetch 函数**（`fetchers/daily_quote.py`）
   - `fetch_tencent_hist(code, market, start_date, end_date, max_k=800)`
   - 处理分段、格式转换、字段映射

2. **新增美股历史日线**（如需要，需先调研数据源）

3. **CLI 入口**（`sync.py`）
   - `python sync.py --type daily-backfill --market CN_A`（A股）
   - `python sync.py --type daily-backfill --market CN_HK`（港股）
   - `python sync.py --type daily-backfill --market all`（全量）
   - `--start-date 20200101`（可选，默认 2020-01-01）
   - `--source tencent|akshare|auto`（可选，默认 auto）

4. **定时任务**（`scheduler.py`）
   - 每天收盘后自动跑增量更新（A股 15:30、港股 16:30）

### 请求策略

- **间隔：** 随机 2~5 秒（不赶时间，优先保证不被限流）
- **超时：** 15 秒/只
- **重试：** 失败 3 次，指数退避（10s → 20s → 40s）
- **边拉边写：** 每 50 只 commit 一次

### 估算时间

| 市场 | 方式 | 估算 |
|------|------|------|
| A 股全量 | 腾讯主源 | ~5,400 × 3.5s = **~5 小时**（含 2 段） |
| 港股全量 | 腾讯主源 | ~2,600 × 3.5s = **~2.5 小时** |
| 日常增量 | 腾讯主源 | ~8,000 × 2s = **~4.5 小时** |

## 风险与应对

| 风险 | 应对 |
|------|------|
| 腾讯限流 | 0.5~1.5s 随机间隔 + fallback 到 akshare |
| 东方财富再次封 IP | 腾讯为主源，akshare 仅作备选 |
| 腾讯接口返回字段不全（无成交额/换手率） | 成交额可从行情快照补充，换手率需计算 |
| 回填过程中断 | 边拉边写 + 断点续传（跳过已有数据） |
